Introduction

This analysis uses MaxDiff, also called Best-Worst Scaling, to measure relative preferences across the ten core MSBA classes. In a MaxDiff task, each respondent sees a small set of alternatives and chooses the class they most prefer and the class they least prefer. The point of the method is to force a clean trade-off: instead of asking respondents to rate every course on a subjective 1–10 scale, MaxDiff asks them to make comparisons.

That matters because rating scales are noisy. One student may treat a 7 as strong approval, while another may treat a 7 as lukewarm. Best-worst choices avoid some of that scale-use bias because the respondent has to separate the strongest option from the weakest option on each screen.

I estimate class preference in three ways. First, I use a simple counts score: the percentage of times a class is chosen best minus the percentage of times it is chosen worst. Second, I fit a multinomial logit model by maximum likelihood. Third, I fit the same multinomial logit model using a Bayesian random-walk Metropolis sampler. The expectation is that all three methods should agree on the broad ranking, especially at the top and bottom. The main difference is that the model-based methods give a more formal choice model, and the Bayesian method makes uncertainty on derived quantities such as preference shares easier to summarize.

The Data

The main data file is maxdiff_design_and_choices.csv. Each row is one item shown to one respondent in one task. The key variables are respondent ID, task number, position on the screen, item number, and response. A response of 1 means the item was chosen as best, -1 means it was chosen as worst, and 0 means it was shown but not selected.

The raw file also contains later two-option rows involving item 11. Because this assignment is about the 10 MSBA class items and the 4-item MaxDiff screens, I filter the data to tasks with exactly four rows and item numbers 1 through 10.

Counts Analysis

The counts analysis is the simplest way to summarize MaxDiff responses. For each class, I compute the percentage of times it was selected as best and subtract the percentage of times it was selected as worst:

\[ \text{score}_j = \%\text{best}_j - \%\text{worst}_j \]

A score near 1 means the class is almost always chosen as best when shown. A score near -1 means the class is almost always chosen as worst. A score near 0 means the class is either in the middle or has mixed reactions.

From MaxDiff Data to MNL Choices

The MaxDiff task can be written as two multinomial logit choice observations. First, the respondent chooses the best class from the four shown classes. If the shown set is \(\{a,b,c,d\}\), the probability that item \(b\) is selected as best is

\[ P(\text{best}=b) = \frac{\exp(\beta_b)}{\exp(\beta_a)+\exp(\beta_b)+\exp(\beta_c)+\exp(\beta_d)}. \]

Then the respondent chooses the worst item from the remaining three items. For the worst choice, the utilities are flipped, so low-utility items become more likely to be chosen:

\[ P(\text{worst}=c \mid \text{best}=b) = \frac{\exp(-\beta_c)}{\sum_{j \in \text{shown} \setminus \{b\}} \exp(-\beta_j)}. \]

The MNL model estimates one utility coefficient \(\beta_j\) for each class. Only differences in utility are identified, so I normalize item 1 to zero and estimate the other nine coefficients relative to item 1.

MNL via Maximum Likelihood

The log-likelihood sums the best-choice and worst-choice log probabilities over all tasks:

\[ \ell(\boldsymbol{\beta}) = \sum_{\text{tasks}} \left[\beta_{j_b^*} - \log \sum_{j \in \text{shown}} \exp(\beta_j) - \beta_{j_w^*} - \log \sum_{j \in \text{shown} \setminus \{j_b^*\}} \exp(-\beta_j)\right]. \]

Here \(j_b^*\) is the chosen best item and \(j_w^*\) is the chosen worst item. I maximize this likelihood using BFGS.

Code 
items = tasks[["item_1", "item_2", "item_3", "item_4"]].to_numpy() - 1
best = tasks["best"].to_numpy() - 1
worst = tasks["worst"].to_numpy() - 1
remaining = np.vstack([row[row != b] for row, b in zip(items, best)])


def full_beta(beta9):
    return np.r_[0, beta9]


def ll(beta9):
    beta = full_beta(beta9)
    best_part = beta[best] - logsumexp(beta[items], axis=1)
    worst_part = -beta[worst] - logsumexp(-beta[remaining], axis=1)
    return np.sum(best_part + worst_part)


out = minimize(lambda b: -ll(b), np.zeros(9), method="BFGS")
beta_hat = full_beta(out.x)
shares = np.exp(beta_hat) / np.exp(beta_hat).sum()
se = np.r_[np.nan, np.sqrt(np.diag(out.hess_inv))]

MNL via Bayesian Estimation

The Bayesian model uses the same MNL likelihood but adds a weakly informative normal prior:

\[ \boldsymbol{\beta} \sim N(\boldsymbol{0}, 10I). \]

The posterior is proportional to the likelihood times the prior:

\[ \pi(\boldsymbol{\beta} \mid \boldsymbol{y}) \propto L(\boldsymbol{\beta})\pi(\boldsymbol{\beta}). \]

I estimate the posterior using a random-walk Metropolis-Hastings sampler. The chain starts at the MLE, runs for 20,000 iterations, and discards the first 25% as burn-in.

Code 
def log_post(beta9):
    prior = np.sum(-0.5 * np.log(2 * np.pi * 10) - beta9**2 / 20)
    return ll(beta9) + prior


def mh(beta0, n_iter, step):
    draws = np.empty((n_iter, len(beta0)))
    beta = beta0.copy()
    lp = log_post(beta)
    accept = 0

    for t in range(n_iter):
        beta_star = beta + np.random.normal(0, step, len(beta0))
        lp_star = log_post(beta_star)

        if np.log(np.random.uniform()) < lp_star - lp:
            beta = beta_star
            lp = lp_star
            accept += 1

        draws[t] = beta

    return draws, accept / n_iter


draws, accept_rate = mh(out.x, n_iter=20000, step=0.08)
burn = 5000
post = draws[burn:]

The MNL model adds value over the counts analysis because it explicitly models the choice process. Each task contributes information not only through the chosen best and worst items, but also through the alternatives that were shown and not selected. The Bayesian model adds an additional layer: uncertainty for nonlinear quantities such as shares of preference is easy to compute by transforming every posterior draw.

Discussion

Substantively, the strongest preference signal is for Marketing Analytics. That is not subtle; it is the top class by a wide margin in all three approaches. Business Analytics, Customer Analytics, and Large Data also perform well. The least preferred classes are Analytics Consulting, Marketing/Finance/Operations, and AI Math/Programming/Analytics. One possible interpretation is that students are more drawn to analytics classes with clear technical or marketing application than to broader foundation or consulting-oriented courses.

Methodologically, the counts analysis is useful as a quick executive summary. It is transparent and easy to explain. The MLE MNL model is better when the goal is formal estimation, model-based shares, and standard errors. The Bayesian MNL model is best when the analysis needs uncertainty intervals for derived quantities or when the model may later be extended to a hierarchical structure with individual-level preferences.

The main caveat is that this survey reflects a self-selected sample of MSBA students, so the results should not be generalized too aggressively. Preferences may also depend on when students took the survey, which classes they had already completed, and how familiar they were with each instructor. With more time, I would estimate respondent-level heterogeneity to see whether the aggregate ranking hides distinct preference segments.

## Introduction

This analysis uses MaxDiff, also called Best-Worst Scaling, to measure relative preferences across the ten core MSBA classes. In a MaxDiff task, each respondent sees a small set of alternatives and chooses the class they most prefer and the class they least prefer. The point of the method is to force a clean trade-off: instead of asking respondents to rate every course on a subjective 1--10 scale, MaxDiff asks them to make comparisons.

That matters because rating scales are noisy. One student may treat a 7 as strong approval, while another may treat a 7 as lukewarm. Best-worst choices avoid some of that scale-use bias because the respondent has to separate the strongest option from the weakest option on each screen.

I estimate class preference in three ways. First, I use a simple counts score: the percentage of times a class is chosen best minus the percentage of times it is chosen worst. Second, I fit a multinomial logit model by maximum likelihood. Third, I fit the same multinomial logit model using a Bayesian random-walk Metropolis sampler. The expectation is that all three methods should agree on the broad ranking, especially at the top and bottom. The main difference is that the model-based methods give a more formal choice model, and the Bayesian method makes uncertainty on derived quantities such as preference shares easier to summarize.

## Counts Analysis

The counts analysis is the simplest way to summarize MaxDiff responses. For each class, I compute the percentage of times it was selected as best and subtract the percentage of times it was selected as worst:

$$
\text{score}_j = \%\text{best}_j - \%\text{worst}_j
$$

A score near 1 means the class is almost always chosen as best when shown. A score near -1 means the class is almost always chosen as worst. A score near 0 means the class is either in the middle or has mixed reactions.

```{python}
counts = (
    data.groupby("Item")
    .agg(
        times_shown=("Item", "size"),
        times_best=("Response", lambda x: (x == 1).sum()),
        times_worst=("Response", lambda x: (x == -1).sum())
    )
    .reset_index()
    .merge(labels, on="Item")
)

counts["pct_best"] = counts["times_best"] / counts["times_shown"]
counts["pct_worst"] = counts["times_worst"] / counts["times_shown"]
counts["counts_score"] = counts["pct_best"] - counts["pct_worst"]
counts["counts_rank"] = counts["counts_score"].rank(ascending=False, method="first").astype(int)
counts = counts.sort_values("counts_rank")

display(counts[["Item", "label", "times_shown", "times_best", "times_worst", "pct_best", "pct_worst", "counts_score", "counts_rank"]].round(3))
```

## From MaxDiff Data to MNL Choices

The MaxDiff task can be written as two multinomial logit choice observations. First, the respondent chooses the best class from the four shown classes. If the shown set is $\{a,b,c,d\}$, the probability that item $b$ is selected as best is

$$
P(\text{best}=b) = \frac{\exp(\beta_b)}{\exp(\beta_a)+\exp(\beta_b)+\exp(\beta_c)+\exp(\beta_d)}.
$$

Then the respondent chooses the worst item from the remaining three items. For the worst choice, the utilities are flipped, so low-utility items become more likely to be chosen:

$$
P(\text{worst}=c \mid \text{best}=b) = \frac{\exp(-\beta_c)}{\sum_{j \in \text{shown} \setminus \{b\}} \exp(-\beta_j)}.
$$

The MNL model estimates one utility coefficient $\beta_j$ for each class. Only differences in utility are identified, so I normalize item 1 to zero and estimate the other nine coefficients relative to item 1.

```{python}
task_rows = []

for (record_id, task), g in data.groupby(["Record ID", "Task"]):

    row = {"Record ID": record_id, "Task": task}

    row["worst"] = g.loc[g["Response"] == -1, "Item"].iloc[0]
```

## MNL via Maximum Likelihood

The log-likelihood sums the best-choice and worst-choice log probabilities over all tasks:

$$
\ell(\boldsymbol{\beta}) = \sum_{\text{tasks}} \left[\beta_{j_b^*} - \log \sum_{j \in \text{shown}} \exp(\beta_j) - \beta_{j_w^*} - \log \sum_{j \in \text{shown} \setminus \{j_b^*\}} \exp(-\beta_j)\right].
$$

Here $j_b^*$ is the chosen best item and $j_w^*$ is the chosen worst item. I maximize this likelihood using BFGS.

```{python}
items = tasks[["item_1", "item_2", "item_3", "item_4"]].to_numpy() - 1
best = tasks["best"].to_numpy() - 1
worst = tasks["worst"].to_numpy() - 1
remaining = np.vstack([row[row != b] for row, b in zip(items, best)])


def full_beta(beta9):
    return np.r_[0, beta9]


def ll(beta9):
    beta = full_beta(beta9)
    best_part = beta[best] - logsumexp(beta[items], axis=1)
    worst_part = -beta[worst] - logsumexp(-beta[remaining], axis=1)
    return np.sum(best_part + worst_part)


out = minimize(lambda b: -ll(b), np.zeros(9), method="BFGS")
beta_hat = full_beta(out.x)
shares = np.exp(beta_hat) / np.exp(beta_hat).sum()
se = np.r_[np.nan, np.sqrt(np.diag(out.hess_inv))]

mle = pd.DataFrame({
    "Item": range(1, 11),
    "beta_hat": beta_hat,
    "se": se,
    "mle_share": shares
}).merge(labels, on="Item")

mle["mle_rank"] = mle["mle_share"].rank(ascending=False, method="first").astype(int)
mle = mle.sort_values("mle_rank")

display(mle[["Item", "label", "beta_hat", "se", "mle_share", "mle_rank"]].round(3))
```

## MNL via Bayesian Estimation

The Bayesian model uses the same MNL likelihood but adds a weakly informative normal prior:

$$
\boldsymbol{\beta} \sim N(\boldsymbol{0}, 10I).
$$

The posterior is proportional to the likelihood times the prior:

$$
\pi(\boldsymbol{\beta} \mid \boldsymbol{y}) \propto L(\boldsymbol{\beta})\pi(\boldsymbol{\beta}).
$$

I estimate the posterior using a random-walk Metropolis-Hastings sampler. The chain starts at the MLE, runs for 20,000 iterations, and discards the first 25% as burn-in.

```{python}
def log_post(beta9):
    prior = np.sum(-0.5 * np.log(2 * np.pi * 10) - beta9**2 / 20)
    return ll(beta9) + prior


def mh(beta0, n_iter, step):
    draws = np.empty((n_iter, len(beta0)))
    beta = beta0.copy()
    lp = log_post(beta)
    accept = 0

    for t in range(n_iter):
        beta_star = beta + np.random.normal(0, step, len(beta0))
        lp_star = log_post(beta_star)

        if np.log(np.random.uniform()) < lp_star - lp:
            beta = beta_star
            lp = lp_star
            accept += 1

        draws[t] = beta

    return draws, accept / n_iter


draws, accept_rate = mh(out.x, n_iter=20000, step=0.08)
burn = 5000
post = draws[burn:]
accept_rate
```

The final acceptance rate is shown above. It falls in the recommended range of roughly 25% to 40%, so the proposal step size is reasonable.

The three methods agree almost perfectly in this dataset. Marketing Analytics is the dominant top class across counts, MLE, and Bayes. Business Analytics and Customer Analytics are consistently second and third, while Analytics Consulting is consistently last. The middle of the ranking is tighter: Capstone and SQL are very close, and small sampling changes could plausibly flip them.

The MNL model adds value over the counts analysis because it explicitly models the choice process. Each task contributes information not only through the chosen best and worst items, but also through the alternatives that were shown and not selected. The Bayesian model adds an additional layer: uncertainty for nonlinear quantities such as shares of preference is easy to compute by transforming every posterior draw.

## Discussion

Substantively, the strongest preference signal is for Marketing Analytics. That is not subtle; it is the top class by a wide margin in all three approaches. Business Analytics, Customer Analytics, and Large Data also perform well. The least preferred classes are Analytics Consulting, Marketing/Finance/Operations, and AI Math/Programming/Analytics. One possible interpretation is that students are more drawn to analytics classes with clear technical or marketing application than to broader foundation or consulting-oriented courses.

Methodologically, the counts analysis is useful as a quick executive summary. It is transparent and easy to explain. The MLE MNL model is better when the goal is formal estimation, model-based shares, and standard errors. The Bayesian MNL model is best when the analysis needs uncertainty intervals for derived quantities or when the model may later be extended to a hierarchical structure with individual-level preferences.

The main caveat is that this survey reflects a self-selected sample of MSBA students, so the results should not be generalized too aggressively. Preferences may also depend on when students took the survey, which classes they had already completed, and how familiar they were with each instructor. With more time, I would estimate respondent-level heterogeneity to see whether the aggregate ranking hides distinct preference segments.
