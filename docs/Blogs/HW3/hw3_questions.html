Poisson Regression and Maximum Likelihood Estimation

A Case Study of Blueprinty’s Software and Patent Awards

Author

Khyati Parihar

Published

May 16, 2026

Introduction

Blueprinty is a small software company that makes blueprint design tools for engineering firms that submit patent applications to the U.S. Patent Office. Its marketing team would like to claim that firms using Blueprinty’s software are more successful in getting patents approved. That sounds appealing, but it is not a simple claim to evaluate.

In an ideal setting, we would observe the same firms before and after they adopted Blueprinty’s software and compare how their patent outcomes changed. We do not have that kind of data here. Instead, we have a cross section of 1,500 mature engineering firms, including each firm’s number of patents awarded over the last five years, its region, its age, and whether it is a Blueprinty customer.

The challenge is that Blueprinty’s customers are not randomly selected. If customer firms are concentrated in certain regions, or if they tend to be older or more established, then raw differences in patent counts may reflect those characteristics rather than the effect of the software itself. Because of that, the analysis below focuses on both careful descriptive comparisons and a regression model that controls for observable differences across firms.

A Simple Poisson Model via Maximum Likelihood

The dependent variable here is a count, the number of patents awarded over the last five years. Counts are non-negative integers, so a Normal model is not a natural starting point. A Poisson model is more appropriate because it is built specifically for outcomes like 0, 1, 2, and so on.

For a single observation, if (Y ()), then the probability mass function is

\[ f(Y \mid \lambda) = \frac{e^{-\lambda}\lambda^Y}{Y!}. \]

If we assume the observations are independent and identically distributed, then the joint likelihood for (Y_1, , Y_n) is

\[ L(\lambda \mid Y_1, \dots, Y_n) = \prod_{i=1}^n \frac{e^{-\lambda}\lambda^{Y_i}}{Y_i!}. \]

Taking logs gives the log-likelihood:

\[ \ell(\lambda) = \sum_{i=1}^n \left( -\lambda + Y_i \log \lambda - \log(Y_i!) \right). \]

This expression is much easier to work with numerically than the raw likelihood because products turn into sums.

To start simply, I estimate a single common () for the whole sample. In this setup, every firm is assumed to have the same expected patent rate.

Y = df["patents"].to_numpy()

def poisson_loglikelihood(lambda_value, Y):
    lambda_value = float(lambda_value)
    if lambda_value <= 0:
        return -np.inf
    return np.sum(-lambda_value + Y * np.log(lambda_value) - gammaln(Y + 1))

Next I plot the log-likelihood over a range of possible λ values.

lambda_grid = np.linspace(0.1, 8, 300)
loglik_values = [poisson_loglikelihood(lam, Y) for lam in lambda_grid]

plt.figure(figsize=(8, 5))
plt.plot(lambda_grid, loglik_values, linewidth=2.5, color="#d96aa7")
plt.axvline(Y.mean(), linestyle="--", linewidth=2, color="#f8b6c8")
plt.xlabel(r"$\lambda$")
plt.ylabel("Log-likelihood")
plt.title("Poisson log-likelihood for a common patent rate")
plt.tight_layout()
plt.show()

The peak of the curve shows the value of λ that best fits the data. Visually, the maximum occurs near the sample mean. That makes sense, because in a Poisson model the mean and the variance are both functions of the same parameter λ.

If we differentiate the log-likelihood and set the derivative equal to zero, the MLE solves to

\[ \hat{\lambda}_{MLE} = \bar{Y} \]

So the analytical MLE is just the sample mean.

I now confirm that numerically.

def negative_poisson_loglikelihood(log_lambda, Y):
    lambda_value = np.exp(log_lambda[0])
    return -poisson_loglikelihood(lambda_value, Y)

simple_result = minimize(
    negative_poisson_loglikelihood,
    x0=np.array([np.log(Y.mean())]),
    args=(Y,),
    method="BFGS"
)

lambda_mle_numeric = np.exp(simple_result.x[0])
lambda_sample_mean = Y.mean()

simple_table = pd.DataFrame({
    "Estimate": [lambda_mle_numeric, lambda_sample_mean]
}, index=["Numerical MLE", "Sample mean"]).round(6)

As expected, the numerical MLE and the sample mean are essentially identical. That is a useful checkpoint because it shows the likelihood function and optimizer are behaving the way they should.

Poisson Regression

The simple model above assumes every firm shares the same patent rate. That is too restrictive for this application. Firms differ in age, region, and customer status, so the expected patent rate should be allowed to vary across firms.

I therefore move to a Poisson regression model:

\[ Y_i \sim \text{Poisson}(\lambda_i) \qquad \text{where} \qquad \lambda_i = \exp(X_i' \beta) \]

This specification lets each firm have its own expected patent count. The exponential link is important because X i ′ ​

β can take any real value, but λ i ​

must always be positive.

I include an intercept, age, age squared, region dummies, and the customer indicator. Age squared is included because the relationship between firm age and patent output is unlikely to be perfectly linear. One region dummy is omitted to avoid perfect collinearity with the intercept.

X = pd.get_dummies(df["region"], prefix="region", drop_first=True, dtype=float)

X = pd.concat(
    [
        pd.Series(1.0, index=df.index, name="const"),
        df["age"].astype(float),
        (df["age"] ** 2).rename("age_sq"),
        X,
        df["iscustomer"].astype(float)
    ],
    axis=1
)

X.columns = [str(col) for col in X.columns]

The regression log-likelihood is \[ \ell(\beta) = \sum_{i=1}^n \left( Y_i X_i' \beta - \exp(X_i' \beta) - \log(Y_i!) \right) \]

I code that below.

X_mat = X.to_numpy()
Y = df["patents"].to_numpy()

def poisson_regression_loglikelihood(beta, Y, X):
    eta = X @ beta
    lambda_i = np.exp(eta)
    return np.sum(Y * eta - lambda_i - gammaln(Y + 1))

def negative_poisson_regression_loglikelihood(beta, Y, X):
    return -poisson_regression_loglikelihood(beta, Y, X)

Now I estimate the coefficients numerically by maximum likelihood.

beta_start = np.zeros(X_mat.shape[1])

poisson_mle = minimize(
    negative_poisson_regression_loglikelihood,
    x0=beta_start,
    args=(Y, X_mat),
    method="BFGS"
)

beta_hat = poisson_mle.x
vcov_approx = poisson_mle.hess_inv
se_hat = np.sqrt(np.diag(vcov_approx))

mle_results = pd.DataFrame({
    "Coefficient": beta_hat,
    "Std. Error": se_hat
}, index=X.columns).round(4)

The customer coefficient is positive, which means Blueprinty customers are predicted to have higher expected patent counts after controlling for age and region. In a Poisson model, coefficients are interpreted on the log scale, so it is often more intuitive to exponentiate them.

The Effect of Blueprinty’s Software

Poisson coefficients are useful, but they are not directly expressed in extra patents. To get a more interpretable quantity, I construct two counterfactual datasets:

  1. one where every firm is treated as a non-customer
  2. one where every firm is treated as a customer

Everything else stays the same. This means each firm keeps its own age and region, and only customer status is changed.

X0 = X.copy()
X1 = X.copy()

X0["iscustomer"] = 0.0
X1["iscustomer"] = 1.0

yhat0 = glm_model.predict(X0)
yhat1 = glm_model.predict(X1)

effect = np.mean(yhat1 - yhat0)

effect_table = pd.DataFrame({
    "Average predicted patents if all were non-customers": [yhat0.mean()],
    "Average predicted patents if all were customers": [yhat1.mean()],
    "Average customer effect": [effect]
}).round(4)

effect_table
Average predicted patents if all were non-customers Average predicted patents if all were customers Average customer effect
0 3.4362 4.229 0.7928

This quantity is easier to explain. It answers the question: if we took the firms in this dataset and changed only their Blueprinty customer status, how much would the model predict their patent counts to change on average?

My estimate implies that being a Blueprinty customer is associated with about 0.79 additional patents over five years, on average, after holding age and region fixed.

Conclusion

This analysis provides evidence that Blueprinty customers tend to receive more patents than non-customers, even after controlling for firm age and region. The raw descriptive comparisons already suggest a customer advantage, and the Poisson regression tells the same story after adjusting for observable differences across firms.

The estimated customer effect is economically meaningful. The model implies that Blueprinty customers have a noticeably higher expected patent rate, and the counterfactual calculation suggests an average increase of roughly 0.79 patents over five years.

At the same time, this is still an observational study. It does not prove that Blueprinty’s software causes the improvement. Customers are not randomly assigned, and there may be unobserved differences between customer and non-customer firms that are not captured by age and region alone. So the careful conclusion is that the results are consistent with Blueprinty’s marketing claim, but they do not establish it with the certainty of a randomized experiment.

```{python}
    return -poisson_loglikelihood(lambda_value, Y)

simple_result = minimize(
    negative_poisson_loglikelihood,
    x0=np.array([np.log(Y.mean())]),
    args=(Y,),
    method="BFGS"
)

lambda_mle_numeric = np.exp(simple_result.x[0])
lambda_sample_mean = Y.mean()

simple_table = pd.DataFrame({
    "Estimate": [lambda_mle_numeric, lambda_sample_mean]
}, index=["Numerical MLE", "Sample mean"]).round(6)

simple_table
```

As expected, the numerical MLE and the sample mean are essentially identical. That is a useful checkpoint because it shows the likelihood function and optimizer are behaving the way they should.

## Poisson Regression

The simple model above assumes every firm shares the same patent rate. That is too restrictive for this application. Firms differ in age, region, and customer status, so the expected patent rate should be allowed to vary across firms.

I therefore move to a Poisson regression model:

$$
Y_i \sim \text{Poisson}(\lambda_i)
\qquad \text{where} \qquad
\lambda_i = \exp(X_i' \beta)
$$

This specification lets each firm have its own expected patent count. The exponential link is important because X
i
′
    ​

β can take any real value, but λ
i
    ​

 must always be positive.

I include an intercept, age, age squared, region dummies, and the customer indicator. Age squared is included because the relationship between firm age and patent output is unlikely to be perfectly linear. One region dummy is omitted to avoid perfect collinearity with the intercept.

```{python}
X = pd.get_dummies(df["region"], prefix="region", drop_first=True, dtype=float)

X = pd.concat(
    [
        pd.Series(1.0, index=df.index, name="const"),
        df["age"].astype(float),
        (df["age"] ** 2).rename("age_sq"),
        X,
        df["iscustomer"].astype(float)
    ],
    axis=1
)

X.columns = [str(col) for col in X.columns]
X.head()
```

The regression log-likelihood is
$$
\ell(\beta) = \sum_{i=1}^n \left( Y_i X_i' \beta - \exp(X_i' \beta) - \log(Y_i!) \right)
$$

I code that below.

```{python}
X_mat = X.to_numpy()
Y = df["patents"].to_numpy()

def poisson_regression_loglikelihood(beta, Y, X):
    eta = X @ beta
    lambda_i = np.exp(eta)
    return np.sum(Y * eta - lambda_i - gammaln(Y + 1))

def negative_poisson_regression_loglikelihood(beta, Y, X):
    return -poisson_regression_loglikelihood(beta, Y, X)
```

Now I estimate the coefficients numerically by maximum likelihood.

```{python}
beta_start = np.zeros(X_mat.shape[1])

poisson_mle = minimize(
    negative_poisson_regression_loglikelihood,
    x0=beta_start,
    args=(Y, X_mat),
    method="BFGS"
)

beta_hat = poisson_mle.x
vcov_approx = poisson_mle.hess_inv
se_hat = np.sqrt(np.diag(vcov_approx))

mle_results = pd.DataFrame({
    "Coefficient": beta_hat,
    "Std. Error": se_hat
}, index=X.columns).round(4)

mle_results
```

To check the result, I fit the same model using statsmodels.

```{python}
glm_model = sm.GLM(Y, X, family=sm.families.Poisson()).fit()

glm_results = pd.DataFrame({
    "Coefficient": glm_model.params,
    "Std. Error": glm_model.bse
}, index=X.columns).round(4)

glm_results
```

Finally, I place the two sets of estimates side by side.

```{python}
comparison = pd.DataFrame({
    "MLE Coef": beta_hat,
    "MLE SE": se_hat,
    "GLM Coef": glm_model.params,
    "GLM SE": glm_model.bse
}, index=X.columns).round(4)

comparison
```

The coefficient estimates match very closely, which is exactly what we would hope to see. The standard errors may differ a bit because statsmodels computes them from its own fitted Hessian, while my hand-coded approach relies on the optimizer’s numerical approximation. That difference is normal and does not undermine the main result.

## Interpreting the coefficients

The customer coefficient is positive, which means Blueprinty customers are predicted to have higher expected patent counts after controlling for age and region. In a Poisson model, coefficients are interpreted on the log scale, so it is often more intuitive to exponentiate them.

```{python}
irr = pd.DataFrame({
    "IRR": np.exp(glm_model.params)
}, index=X.columns).round(4)

irr
```

The incidence rate ratio for iscustomer is above 1, which means customer status is associated with a higher expected patent rate. Specifically, the coefficient implies that customers have an expected patent rate that is about 23 percent higher than otherwise similar non-customers, holding age and region fixed.

The age terms are also sensible. The linear age coefficient is positive while the squared term is negative, which suggests patenting tends to rise with age at first but eventually levels off. That is a plausible pattern for mature firms. The region coefficients show that firms in some regions appear more patent-intensive than others, which supports the earlier concern that geography is an important confounder.

## The Effect of Blueprinty's Software

Poisson coefficients are useful, but they are not directly expressed in extra patents. To get a more interpretable quantity, I construct two counterfactual datasets:

1. one where every firm is treated as a non-customer
2. one where every firm is treated as a customer

Everything else stays the same. This means each firm keeps its own age and region, and only customer status is changed.

```{python}
X0 = X.copy()
X1 = X.copy()

X0["iscustomer"] = 0.0
X1["iscustomer"] = 1.0

yhat0 = glm_model.predict(X0)
yhat1 = glm_model.predict(X1)

effect = np.mean(yhat1 - yhat0)

effect_table = pd.DataFrame({
    "Average predicted patents if all were non-customers": [yhat0.mean()],
    "Average predicted patents if all were customers": [yhat1.mean()],
    "Average customer effect": [effect]
}).round(4)

effect_table
```

This quantity is easier to explain. It answers the question: if we took the firms in this dataset and changed only their Blueprinty customer status, how much would the model predict their patent counts to change on average?

My estimate implies that being a Blueprinty customer is associated with about 0.79 additional patents over five years, on average, after holding age and region fixed.

## Conclusion

This analysis provides evidence that Blueprinty customers tend to receive more patents than non-customers, even after controlling for firm age and region. The raw descriptive comparisons already suggest a customer advantage, and the Poisson regression tells the same story after adjusting for observable differences across firms.

The estimated customer effect is economically meaningful. The model implies that Blueprinty customers have a noticeably higher expected patent rate, and the counterfactual calculation suggests an average increase of roughly 0.79 patents over five years.

At the same time, this is still an observational study. It does not prove that Blueprinty’s software causes the improvement. Customers are not randomly assigned, and there may be unobserved differences between customer and non-customer firms that are not captured by age and region alone. So the careful conclusion is that the results are consistent with Blueprinty’s marketing claim, but they do not establish it with the certainty of a randomized experiment.
