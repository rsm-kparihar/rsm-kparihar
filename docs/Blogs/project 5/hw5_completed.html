Objective

This analysis identifies and quantifies the key drivers of customer satisfaction using the methods from the Week 9 Key Drivers lecture. The response variable is customer satisfaction, and the predictors are nine credit-card perception measures.

The goal is to reproduce the structure of the key-drivers comparison table from the lecture and compare how different variable-importance methods rank the same attributes.

Setup

Code 
import pandas as pd
import numpy as np
import itertools
from math import factorial
from sklearn.linear_model import LinearRegression
from sklearn.preprocessing import StandardScaler
from sklearn.ensemble import RandomForestRegressor

pd.set_option("display.max_columns", None)

pink_style = [
    {"selector": "thead th", "props": [("background-color", "#fce7f3"), ("color", "#9d174d")]},

label_map = {

    "build": "Helps build credit quickly",

    "easy": "Is easy to use",

}

X = df[x_cols].copy()

Methods

The analysis uses six variable-importance approaches. The lecture slide also shows polychoric correlations, but this Python version reports Pearson correlations because the provided assignment can be completed without a dedicated polychoric-correlation package:

  1. Pearson correlations: R² from a simple regression (correlation squared), normalized to sum to 100%.
  2. Standardized regression coefficients: absolute standardized multiple-regression coefficients, normalized to sum to 100%.
  3. Drop-one usefulness: decrease in full-model (R^2) after removing each driver, normalized to sum to 100%.
  4. LMG / Shapley values: each driver’s average marginal contribution to (R^2) across all possible model-entry orders.
  5. Johnson’s relative weights: a fast linear-regression approximation to Shapley values.
  6. Random Forest importance: impurity-based feature importance from a random forest model.

Calculations

Code 
def pct(s):
    s = pd.Series(s, index=x_cols).astype(float)
    total = s.sum()
    if total == 0:
        return s * np.nan
    return 100 * s / total

# Pearson: R² from simple regression (Slide 64)
pearson = pd.Series({c: X[c].corr(y)**2 for c in x_cols})
pearson_pct = pct(pearson)

# Standardized regression coefficients: absolute standardized betas, normalized
scaler_x = StandardScaler()
scaler_y = StandardScaler()
Xs = scaler_x.fit_transform(X)
ys = scaler_y.fit_transform(y.to_numpy().reshape(-1, 1)).ravel()

lm_std = LinearRegression().fit(Xs, ys)
std_beta = pd.Series(lm_std.coef_, index=x_cols)
std_beta_pct = pct((std_beta.abs())**2)

# Drop-one usefulness: full R2 minus R2 without each driver, normalized
full_model = LinearRegression().fit(X, y)
full_r2 = full_model.score(X, y)

drop_one = {}
for c in x_cols:
    X_minus = X.drop(columns=[c])
    r2_minus = LinearRegression().fit(X_minus, y).score(X_minus, y)
    drop_one[c] = full_r2 - r2_minus

drop_one = pd.Series(drop_one)
drop_one_pct = pct(drop_one)

# Shapley / LMG values using R2 as the fit statistic
r2_cache = {}

def model_r2(features):
    features = tuple(sorted(features))
    if len(features) == 0:
        return 0.0
    if features not in r2_cache:
        r2_cache[features] = LinearRegression().fit(X[list(features)], y).score(X[list(features)], y)
    return r2_cache[features]

n = len(x_cols)
shapley = pd.Series(0.0, index=x_cols)

for c in x_cols:
    others = [x for x in x_cols if x != c]
    for k in range(n):
        weight = factorial(k) * factorial(n - k - 1) / factorial(n)
        for subset in itertools.combinations(others, k):
            shapley[c] += weight * (model_r2(subset + (c,)) - model_r2(subset))

shapley_pct = pct(shapley)

# Johnson relative weights
R = np.corrcoef(X, rowvar=False)
rxy = np.array([np.corrcoef(X[c], y)[0, 1] for c in x_cols])

eigval, eigvec = np.linalg.eigh(R)
eigval = np.clip(eigval, 0, None)
L = eigvec @ np.diag(np.sqrt(eigval)) @ eigvec.T
beta_star = np.linalg.pinv(L) @ rxy
johnson = pd.Series((L ** 2) @ (beta_star ** 2), index=x_cols)
johnson_pct = pct(johnson)

# Random Forest importance
rf = RandomForestRegressor(
    n_estimators=1000,
    random_state=495,
    max_features="sqrt"
)
rf.fit(X, y)
rf_pct = pct(pd.Series(rf.feature_importances_, index=x_cols))

results = pd.DataFrame({
    "Perception": [label_map[c] for c in x_cols],
    "Pearson Correlations": pearson_pct.values,
    "Standardized Multiple Regression Coefficients": std_beta_pct.values,
    "Drop-One Usefulness": drop_one_pct.values,
    "LMG / Shapley Values": shapley_pct.values,
    "Johnson's Epsilon": johnson_pct.values,
    "Mean Decrease in RF Gini Coefficient": rf_pct.values
}, index=range(1, len(x_cols) + 1))

results_display = results.copy()

    results_display[c] = results_display[c].map(lambda x: f"{x:.1f}%")

results_display.style.set_caption("Key Drivers of Customer Satisfaction").set_table_styles(pink_style)

Overall, the key-drivers analysis suggests that satisfaction is most strongly associated with a small set of core perceptions. The exact ranking changes somewhat by method, which is expected because each method defines importance differently. Pearson correlations measure one-variable-at-a-time association, drop-one usefulness measures incremental value after controlling for all other variables, Shapley values average incremental value over all possible model orderings, Johnson’s method approximates Shapley values for linear regression, and random forests allow nonlinear tree-based relationships.

Method Comparison

Pearson correlations are easy to interpret, but they can overstate importance when drivers are correlated with each other. Drop-one usefulness and standardized regression coefficients can understate importance when predictors overlap. Shapley values solve this by allocating shared explanatory power across variables, but they require estimating many models. Johnson’s relative weights provide a fast approximation to Shapley values for linear regression. Random forests are more flexible because they are not restricted to a linear structure, but their impurity-based importance values should be interpreted as model-based importance rather than causal effects.

Conclusion

This analysis replicates the key-drivers framework from the lecture using the provided customer satisfaction survey data. The most reliable drivers are the ones that remain highly ranked across Shapley values, Johnson’s relative weights, and random forest importance. These attributes should receive the most attention when deciding which customer perceptions to prioritize for improving satisfaction.
